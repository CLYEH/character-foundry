## 0. Design Tokens（給 Frontend + Design Agent 參考起點）

非正式 design system，僅給出方向：

```
Typography:
  Heading:   Noto Sans TC / 系統字 (粗)
  Body:      Noto Sans TC / 系統字 (regular)
  Mono:      JetBrains Mono (prompt preview / request_id 用)

Spacing scale:
  4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 (px)

Color intent (具體色票由 Design Agent 定):
  Primary    -> 用於主要 CTA (建立、生成、下載)
  Danger     -> 刪除 / 錯誤
  Warning    -> 冷靜提醒 (quota 接近上限、degraded mode)
  Success    -> Toast / 完成狀態
  Neutral    -> 背景 / 分隔 / 次要文字

Radius:
  Card:      8px
  Button:    6px
  Input:     6px
  Modal:     12px
```

**Breakpoints（Phase 1 主要桌面）：**
- `>= 1280px`：完整佈局
- `1024-1279px`：側欄收合
- `< 1024px`：Phase 1 只支援最基礎 layout（平板可用，手機體驗非重點）

## 1. 全域元件

### 1.1 Top Nav（所有頁面固定）

```
┌────────────────────────────────────────────────────────────────────────┐
│ 🎭 CHARACTER FOUNDRY    🔍 搜尋角色...     📊 本月 $42.5  👤 Alice ▾    │
└────────────────────────────────────────────────────────────────────────┘
```

- Logo（左）：點擊回 `/`
- Search（中）：全文搜尋 Character name（用 pg_trgm 支援模糊）
- Usage widget（右）：顯示當月累積成本，點擊 → `/usage`
- User menu（最右）：下拉 = 設定 / 登出

### 1.2 Toast 通知（右下角浮現，stackable）

```
                                           ┌──────────────────────┐
                                           │ ✓ 紅旗袍版 生成完成    │
                                           │ 點擊查看               │
                                           └──────────────────────┘

                                           ┌──────────────────────┐
                                           │ ✗ 招手動作 生成失敗    │
                                           │ 原因：模型忙碌中       │
                                           │ [詳細] [重試] [關閉]   │
                                           └──────────────────────┘
```

### 1.3 全域 Banner（degraded mode / 嚴重警告）

位於 Top Nav 下方，平時隱藏：

```
┌────────────────────────────────────────────────────────────────┐
│ ⚠ Prompt 最佳化暫時降級，品質可能稍降。預計 5 分鐘後恢復。      │
└────────────────────────────────────────────────────────────────┘
```

觸發條件：circuit breaker OPEN、reconciler degraded、外部 API 異常。

**狀態顯示：**

| Checkpoint 狀態 | 顯示 |
|---|---|
| Queued | `🕒 #2 in queue` |
| Running (有 progress) | Progress bar 覆蓋縮圖下方 |
| Running (無 progress) | 縮圖位置跑 indeterminate spinner + `預計還要 12 秒` |
| Completed | 縮圖 + `[這張] [用這張再改]` |
| Failed | `⚠ 失敗` + 重試按鈕 |
| Cancelled | 灰色縮圖 + `已取消` |

**用這張再改**：點擊某 checkpoint 的「用這張再改」→ 該 checkpoint 的選單設定 + 補述載入到左邊輸入區，使用者可以微調再點「生成新候選」。

**[選為 Base 並完成]**：按鈕永遠可見，但未選中 checkpoint 時 disabled。點擊 → 進確認 modal → API `select-base` → 跳 P-05。

**離開建立**：點「取消建立」→ 確認 modal：「此 session 會保留 7 天，checkpoint 還可回來挑」→ nav 離開。

## 15. 取消中間態（Backend U4 反映）

使用者在 Task running 時按「取消」：

```
┌──────────────────────────────┐
│ ⏳ 取消中...                 │
│                              │
│ 🟡 如果模型已開始回應，      │
│    此動作可能仍會完成        │
│ [強制關閉 (可能成本已計入)]   │
└──────────────────────────────┘
```

最終：
- Worker 成功 abort → Toast `已取消`
- Worker 來不及 → Toast `任務已完成 (取消未成功)` + 結果仍會出現
